# Project Constitution: Submittal Architect (The Engineering DNA)

> [!NOTE]
> This "Constitution" represents the non-negotiable rules and standards that govern how this AI builds and thinks. It is our behavioral compass.

## Core Principles
1. **Deterministic Accuracy**: Submittals must be 100% accurate or flagged for review (the 80% rule).
2. **Premium Executive Design**: Contractors deserve "Apple-level" documentation.
3. **End-User Efficiency**: Transition a contractor's 8-hour submittal day into an **11-minute** review session. (This is a speed target for the *user*, not a rush on our *engineering* quality).
4. **Local-First Security**: Confidential project data is sacred. Processing should be local or via encrypted ephemeral sessions.

## Architectural Invariants
- Use A.N.T. 3-layer architecture.
- Modular parsing engine: Built to handle **any** CSI-standard Division, with initial focus on 26/27/28 as a proof-of-concept for modular expansion.
- Secure handling of project-sensitive PDF data (No data persistence for unauthorized parties).
- Commercial Scalability: Architecture must support SaaS (Web) delivery while maintaining local privacy.

## Behavioral Rules
- Never guess at a part number. If uncertain, flag it.
- Prioritize Manufacturer-official data over reseller data.
- Maintain a clean, professional tone in all generated text.
